# Story 4.5: Client Settings Management

**Epic:** 4 - Administration & Settings
**Story Key:** 4-5-client-settings
**Status:** pending

## 1. User Story

**As an** Admin / Ops Manager,
**I want** to manage a comprehensive list of Clients,
**So that** I can maintain accurate records (TIN, System ID) and standardise client selection across the system.

## 2. Acceptance Criteria (BDD)

### Scenario 1: Client Registry List
**Given** I am on the "Client Settings" page
**When** I view the registry
**Then** I see a list of clients with:
*   **System ID** (Internal Reference)
*   **Client Name**
*   **TIN** (Tax Identification Number)

### Scenario 2: Add New Client
**Given** I open the "Add Client" modal
**When** I enter the **Client Name** and **TIN**
**Then** a **System ID** is auto-generated (Frontend Logic) for internal reference
**And** I can save the new client to the system.

### Scenario 3: Waybill Integration
**Given** I am creating a new Waybill
**When** I reach the Client field
**Then** I must select from the existing Client list (Searchable Select)
**And** I can no longer enter a free-text client name.

## 3. Technical Requirements

### 🏗️ Data Models

```python
class Client(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    system_id: str = Field(unique=True, index=True) # Generated by Frontend
    name: str = Field(index=True)
    tin: str | None = Field(default=None) # Optional
    created_at: datetime = Field(default_factory=datetime.utcnow)
    updated_at: datetime = Field(default_factory=datetime.utcnow)
```

### 🔗 Integration
*   **API:** Standard CRUD endpoints (`/api/v1/clients`).
*   **Frontend:**
    *   `src/app/settings/clients/page.tsx`: Management Interface.
    *   `src/components/waybills/CreateWaybillDrawer.tsx`: Update to use `Client` select.
